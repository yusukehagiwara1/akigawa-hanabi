# 秋川流域花火大会 2026 — Web担当としての目標設定と取り組み案

> Web構築担当（萩原）として、2026年全体目標「来場者 8,000人」達成のために
> 設定する目標・役割・実行プラン。MTG 用および自身の活動指針として作成。

## 0. 結論（先出し）

> **2026年の全体目標「8,000人」達成のため、Web担当としては「来場意思決定ファネルの設計・最適化責任者」として、サイト経由のチケット販売貢献を前年比3倍に引き上げる。**

---

## 1. なぜ昨年は 8,000人 を達成できなかったか（Web担当範囲の考察）

戦略書にある通り、SNS の役割は「情報発信ではなく**来場意思決定の後押し**」です。同じことが Web サイトにも言えます。前年の旧WPサイトを振り返ると、**「意思決定を後押しする仕組みが弱かった」** ことが最大の課題でした。

| 想定されたボトルネック | 旧サイトの実態 | 影響 |
|---|---|---|
| ① **検索流入の弱さ** | タイトル・description が「秋川流域花火大会」のみで「あきる野 花火」「東京 秋 花火」「紅葉 花火」等の指名外キーワードでは上位に出ない | 認知段階で取りこぼし |
| ② **比較段階の情報不足** | アクセス・駐車場・混雑度といった「他の花火大会と比較する人」が知りたい情報が散在 | 検討離脱 |
| ③ **CTA が弱い** | チケット購入導線が深い階層、SP では完全に埋もれていた | 来場転換 CVR が低い |
| ④ **計測がない** | GA4 未導入。どこで離脱しているか不明 → 改善が経験則ベース | PDCA が回らない |
| ⑤ **継続接点ゼロ** | 一度見て帰った人を呼び戻す手段がない（メール登録/LINE 等なし） | 再訪離脱が多い |
| ⑥ **社会的証明の不足** | 体験者の声・メディア掲載・過去画像が静的でストーリーを語らない | 安心感の醸成が弱い |
| ⑦ **モバイル UX** | レスポンシブが甘く、SP からの離脱率が高かった可能性 | 主要流入である SP で機会損失 |

→ つまり「来場したい人が情報を取り切れなかった」「迷っている人の背中を押せなかった」**意思決定の摩擦が大きすぎた**ことが、Web 起因の主因だと考えます。

## 2. Web担当としての役割の再定義（より具体的に）

| 現在の役割（ざっくり） | 提案する具体定義 |
|---|---|
| サイトを作る人 | **来場意思決定ファネルの設計・最適化責任者**<br>＝ 認知→比較→来場転換→拡散の4段階を、サイト上で最大化する役割 |

具体的には以下 4 つの責任を持ちます：

1. **計測責任**：GA4 ファネル設計・月次レポートを運営チームに提供
2. **検索流入責任**：SEO・コンテンツ設計で指名外キーワードからの新規流入を獲得
3. **CVR 責任**：チケット販売 / 寄付協賛 / メール登録のコンバージョン率を最大化
4. **継続接点責任**：メール・LINE・PWA Push で「一度見た人を逃さない」基盤を構築

## 3. 個人目標（2026年 来場 8,000人 達成までの KPI）

戦略書の SNS KPI（Instagram 1万、UGC 500件、SNS経由来場 20%）と整合する形で、**Web 側の責任 KPI** を設定しました。

| 指標 | 2025年（推定/実績） | 2026年目標 | 達成手段 |
|---|---|---|---|
| **① サイト訪問 UU（年間）** | 約 20,000 | **60,000**（3倍） | SEO + SNS連携強化 |
| **② チケット購入クリック UU** | 計測なし | **6,000**（CVR 10%） | CTA最適化 + 比較情報強化 |
| **③ メール通知登録者数** | 0 | **500件** | 次回開催情報LP（実装済）の運用 |
| **④ ふるさと納税クリック** | 計測なし | **300件** | satofull 導線の常設化 |
| **⑤ 寄付/協賛フォーム送信** | 計測なし | **50件** | 寄付・協賛LP の改善 |
| **⑥ モバイルLCP（速度）** | 不明（おそらく3秒+） | **< 2.5秒** | 画像最適化・SW・PWA |
| **⑦ "東京 秋 花火" 検索順位** | 圏外 | **Top 3** | SEO設計・コンテンツ拡充 |
| **⑧ JSエラー率** | 不明 | **< 0.1%** | エラーモニタリング（実装済） |

→ ①〜⑤ がそのまま「サイト経由の来場貢献」を可視化します。**SNS経由来場 20% を支える「受け皿」がサイト**であり、ここの CVR を上げることが、SNS の頑張りを成果に変える鍵です。

## 4. 具体的な取り組みアイデア（実行プラン）

### Phase 1 — 基盤整備（〜2026年4月）✅ 一部完了

- ✅ GA4 導入＋イベント計測（ticket / donation / sponsor / furusato / SNS）
- ✅ メール通知 LP 実装（次回開催情報を逃さないため）
- ✅ JS エラーモニタリング（離脱の隠れた原因を発見）
- ✅ Service Worker / PWA 対応（再訪時の体験向上）
- ✅ SEO 強化（タイトル・description・JSON-LD・sitemap）
- ⏳ **Google フォーム × メール登録の運用開始**（運営側の作業）
- ⏳ **LINE 公式アカウントの開設と連携**

### Phase 2 — 検討離脱を防ぐ（5月〜7月：認知期）

- **A. 比較されるための情報を充実**
  - 「東京の秋花火大会 比較」を意識した独自コンテンツ
  - 駐車場あり・渋滞少ない・SP席の快適さなど "差別化ポイント" を見せる LP セクション
- **B. 検索流入の拡大**
  - 「紅葉と花火」「秋花火 デート」「子連れ 花火 東京」などのキーワード対応ページ
  - 周辺観光（akiruno-kanko.html）と連動した記事化
- **C. 過去体験者の声 (testimonial CMS) 運用開始**
  - 戦略書の「共感投稿」を Web に持ち込み、安心感を醸成

### Phase 3 — 来場転換を上げる（8月〜10月：比較・転換期）

- **D. チケットページの A/B テスト**
  - SS席 / A席 / フリーエリアの選び方ガイド
  - ファーストビューの CTA 配置検証
- **E. 「行く理由」をビジュアル化**
  - 戦略書の "余白・安心・体験" を伝えるヒーロー演出
  - 「都内から1時間でこの景色」のような戦略書バズ台本を Web でも採用
- **F. SNS → サイトのスムーズな受け皿**
  - SNS 投稿から飛んできた人が直感的にチケットに辿り着く動線
  - UGC（Instagram 投稿）の自動埋め込み

### Phase 4 — 拡散と次年度準備（11月：開催月〜）

- **G. リアルタイム情報の自動更新**
  - 雨天判断・当日案内をサイトTOPに即時掲出
- **H. 開催後の体験記事 / 写真ギャラリー**
  - 「来年も行きたい」と思わせる事後 UX → 次年度のメール登録への循環
- **I. 月次レポートの定例化**
  - GA4 → Looker Studio で運営全員が数値を見られる状態に
