# Looker Studio ダッシュボード セットアップ

GA4 のデータを Looker Studio で可視化し、運営チームが毎週・毎月の進捗を 5 分で把握できる状態にする。

## なぜ Looker Studio？

- **無料**（Google アカウントで使える）
- GA4 と **ネイティブ統合**（追加のスクリプト不要）
- 共有 URL でチームに配信可能（権限管理あり）
- スマホでも閲覧可能

## 想定読者

- 運営チーム（数値を見て意思決定する人）
- Web 担当（初回セットアップする人）

---

## STEP 1: 接続設定（初回のみ、約 10 分）

### 1-1. Looker Studio にログイン

1. https://lookerstudio.google.com/ にアクセス
2. GA4 と同じ Google アカウントでログイン
   - **重要**: GA4 プロパティへの閲覧権限があるアカウントを使用

### 1-2. データソース接続

1. 右上の「**＋ 作成**」→「データソース」
2. コネクタ一覧から「**Google アナリティクス**」を選択
3. アカウント・プロパティを選択
   - アカウント：（NPO法人まちづくりコンソーシアム のもの）
   - プロパティ：**秋川流域花火大会**（GA4プロパティID: `536964506`）
   - データストリーム：Web
4. 右上の「**接続**」をクリック → フィールド一覧が表示される
5. 「**レポートを作成**」をクリック

### 1-3. データソース名を分かりやすく変更

- 上部のデータソース名を「秋川花火 GA4」に変更（任意）

## STEP 2: 必須チャート 5 つを配置

戦略書の KPI に合わせて、最も重要な 5 つの指標を可視化する。

### チャート 1: サイト訪問 UU（時系列折れ線）

**目的**: 「サイト訪問者数が増えているか」を一目で確認

| 項目 | 値 |
|---|---|
| グラフタイプ | 時系列グラフ（折れ線） |
| ディメンション | 日付 |
| 指標 | アクティブユーザー（totalUsers） |
| 期間 | 過去 28 日（デフォルト） |
| 比較期間 | 前期間 ON |
| タイトル | 「日次UU推移」 |

### チャート 2: 主要コンバージョン（円グラフ）

**目的**: 「どのコンバージョンが多いか」の構成比

| 項目 | 値 |
|---|---|
| グラフタイプ | 円グラフ（ドーナツ） |
| ディメンション | イベント名（eventName） |
| 指標 | イベント数（eventCount） |
| フィルタ | `eventName` IN (`ticket_purchase_click`, `donation_click`, `sponsor_click`, `furusato_click`, `sns_click`) |
| タイトル | 「コンバージョン構成」 |

### チャート 3: チケット販売の販売店別（積み上げ棒）

**目的**: 「KKday と アソビュー、どちらが効いているか」

| 項目 | 値 |
|---|---|
| グラフタイプ | 積み上げ棒グラフ |
| ディメンション（X軸） | 日付 |
| 内訳ディメンション | カスタムイベントパラメータ「provider」（`kkday` / `asoview`） |
| 指標 | `ticket_purchase_click` のイベント数 |
| タイトル | 「チケット購入クリック（販売店別）」 |

> **設定メモ**: GA4 のカスタムパラメータ「provider」を Looker Studio で使うには、GA4 側でカスタムディメンションとして登録が必要。GA4管理 → カスタム定義 → 「カスタムディメンションを作成」→ パラメータ名: `provider`、範囲: イベント

### チャート 4: 流入元（テーブル）

**目的**: 「どこから訪問者が来ているか」

| 項目 | 値 |
|---|---|
| グラフタイプ | テーブル |
| ディメンション | デフォルトチャネルグループ（sessionDefaultChannelGroup） |
| 指標 1 | セッション |
| 指標 2 | アクティブユーザー |
| 指標 3 | エンゲージメント率 |
| 並び替え | セッション 降順 |
| 表示行数 | 10 |
| タイトル | 「流入チャネル」 |

戦略書の「SNS経由来場 20%」の検証に直結。

### チャート 5: デバイス別（円グラフ）

**目的**: 「モバイル vs PC 比率」

| 項目 | 値 |
|---|---|
| グラフタイプ | 円グラフ |
| ディメンション | デバイスカテゴリ（deviceCategory） |
| 指標 | セッション |
| タイトル | 「デバイス比率」 |

## STEP 3: スコアカード（数値だけ大きく）

ダッシュボード上部に「**今期の重要指標**」をスコアカードで表示：

1. **UU（過去 28 日）**
   - 指標: アクティブユーザー
   - 比較: 前期間 ON
2. **ticket_purchase_click（過去 28 日）**
   - 指標: イベント数 / フィルタ: `eventName = ticket_purchase_click`
3. **メール登録 LP クリック数**
   - 指標: イベント数 / フィルタ: `cta_label CONTAINS "登録"` または `eventName = sign_up`
4. **平均エンゲージメント時間**
   - 指標: 平均エンゲージメント時間
5. **直帰率**
   - 指標: 直帰率

## STEP 4: フィルタ・コントロールを追加

ダッシュボード上部に「**期間選択フィルタ**」を配置：

1. 上部メニュー「コントロールを追加」→「期間設定」
2. デフォルト期間：過去 28 日
3. 配置：左上

これでユーザーが「今月だけ見たい」「半年で見たい」と切り替え可能。

---

## STEP 5: 共有設定

### 運営チームへ閲覧権限を付与

1. 右上の「**共有**」ボタン
2. メールアドレスを追加
3. 権限：「**閲覧者**」
4. 「リンクをコピーして送信」

### URL を docs/README.md に記録

サイト管理上、URL をどこかに残しておく：

```markdown
- Looker Studio ダッシュボード: https://lookerstudio.google.com/...
```

---

## STEP 6: 自動配信（任意）

毎週月曜の朝にメールで自動配信する設定：

1. 右上の「**配信**」→「メール配信のスケジュール」
2. 頻度：毎週月曜
3. 時刻：09:00 JST
4. 宛先：運営チームのメール
5. 形式：PDF

これで「ダッシュボードを見に行く」必要すらなくなる。

---

## STEP 7: テンプレート活用（時短）

ゼロから作るのが大変なら、Google 公式テンプレートを流用：

1. https://lookerstudio.google.com/gallery にアクセス
2. 「**GA4 Acquisition Report**」を選択
3. 右上「**テンプレートを使用**」→ データソースを「秋川花火 GA4」に紐付け
4. 上記 STEP 2 の 5 つのチャートを追加でカスタマイズ

## 動作確認チェックリスト（QA）

セットアップ完了後、以下を確認：

- [ ] ダッシュボードに今日の UU が表示される（リアルタイムで見えなければ 24 時間後に再確認）
- [ ] 期間フィルタを変更すると全チャートが更新される
- [ ] スコアカード 5 つが正しい値を表示
- [ ] スマホからアクセスしても見られる
- [ ] 運営メンバーに共有して見られることを確認
- [ ] PDF 配信が機能する（テストで自分宛に送信）

## トラブルシューティング

### Q1. データが「0」しか出ない
A: GA4 のデータは 24-48 時間遅延することがあります。1-2 日後に再確認。

### Q2. 「provider」が選べない
A: GA4 でカスタムディメンション登録が必要：
1. GA4 → 管理 → カスタム定義
2. 「カスタムディメンションを作成」
3. パラメータ名: `provider`、範囲: イベント
4. 登録 24-48 時間後に Looker Studio で見える

### Q3. 権限エラー
A: GA4 プロパティへの閲覧権限と、Looker Studio へのログインアカウントが同一か確認。
